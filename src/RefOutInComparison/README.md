## نتیجه‌گیری کوتاه
- برای استراکچرهای بزرگ که فقط نیاز به خواندن دارند، in بهترین گزینه می‌باشد.
- برای تغییر سریع اما با ریسک تغییر ناخواسته ref بهترین گزینه می‌‌باشد.
- برای بازگردانی مقدار جدید با سرعت بالا out بهترین گزینه می‌باشد.
- برای کپی استراکچرهای سنگین روش معمول مناسب نمی‌باشد.
